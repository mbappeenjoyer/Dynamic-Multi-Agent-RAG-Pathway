Supervisor Agent¶

Overview¶

The Supervisor Agent is the governing agent designed to handle user queries, leveraging tools, descriptions, and a RAG from a knowledge base. It supports iterative problem-solving, manages errors, and facilitates follow-up interactions with users. This agent ensures robust handling of tool calls and dynamically resolves execution errors.

Initialization¶

class SUPERVISOR_AGENT(tools, tools_aux, llm, tool_map, path, reflextion_limit=10, top_k=5, max_steps=10)¶

Initialize the SUPERVISOR_AGENT with the necessary tools, retrieval systems, and memory setup.

Parameters:

  • tools (list) – Primary tools available for query resolution.

  • tools_aux (list) – Auxiliary tools for fallback or extended processing.

  • llm (object) – The language model used for generating responses.

  • tool_map (dict) – A mapping of tool names to their descriptions.

  • path (str) – File path for the PDF.

  • reflextion_limit (int, optional) – The maximum attempts to modify agent_code and resolve the error.

  • top_k (int, optional) – Number of top results to retrieve from retriver.

  • max_steps (int, optional) – The maximum attempts allowed for the RAG agent to determine the final answer.

Methods¶

run(query, follow_up_response=None)¶

Generate an answer to the given query, utilizing previous responses for context and prompting follow-up questions.

Parameters:

  • query (str) – The user’s question.

  • follow_up_response (list) – (Optional) List of responses from previous queries for additional context.

Returns:

The answer to the user’s query.

Return type:

str

api_reflexion(agent_code)¶

Manage faulty tool calls by either modifying or removing a tool. This mechanism ensures optimal tool usage.

Parameters:

agent_code (list) – The problematic tool call, structured as: [“tool_name”, [arguments], “Reasoning”]

Returns:

A tuple containing the modified tool call and the response generated after modification.

Return type:

tuple

code_reflexion(agent_code, error)¶

Resolve Python execution errors during tool calls by modifying the tool’s code.

Parameters:

  • agent_code (list) – The last tool call that caused the error.

  • error (str) – The error message generated during execution.

Returns:

The modified tool call and its response.

Return type:

tuple

build_code()¶

Retrieve the tool called and its response.

Returns:

The tool call and its associated response.

Return type:

tuple

detect_failure(agent_code, callback)¶

Detect the type of error in a tool call and categorize its severity.

Parameters:

  • agent_code (list) – The erroneous tool call.

  • callback (str) – The error message or code.

Returns:

“1” if the error is significant, “0” otherwise.

Return type:

str

critic_agent(agent_code, desc, func_response, scratchpad)¶

Evaluate the correctness of tool call arguments and their alignment with the query.

Parameters:

  • agent_code (list) – The current tool call.

  • desc (str) – A description of the tool call’s purpose.

  • func_response (str) – The response from the tool call.

  • scratchpad (str) – Context from prior tool calls.

Returns:

A list containing the evaluation result and reasoning.

Return type:

list

silent_reflexion(code, reason)¶

Resolve invalid tool call arguments flagged by the critic_agent.

Parameters:

  • code (str) – The tool call with potentially invalid arguments.

  • reason (str) – Explanation of why the arguments are invalid.

Returns:

A corrected tool call with valid arguments.

Return type:

list

Error Handling¶

The Supervisor Agent employs robust mechanisms to detect and resolve errors: * API Reflection: Identifies and handles missing or misconfigured tools. * Code Reflection: Fixes execution errors in tool calls. * Silent Reflection: Adjusts invalid arguments in tool calls.

Requirements¶

  • langchain

  • llama_index

  • jinaai for embeddings

  • A compatible LLM for query generation

Recommended Usage¶

The following demonstrates how to initialize and use the SUPERVISOR_AGENT:

# Initialize the agent
agent = SUPERVISOR_AGENT(tools, tools_aux, llm, tool_map, "path/to/pdf")

# Process a query
answer = agent.run("What is the net revenue of the company and find 5% of it?")
print(answer)
