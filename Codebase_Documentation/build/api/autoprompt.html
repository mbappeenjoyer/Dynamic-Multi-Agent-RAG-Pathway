Autoprompt¶

Overview¶

Autoprompt is an intent-based prompt calibration algorithm that is utilized to dynamically generate tools to cater to various user needs. The tools are essentially designed to generate python code to solve the task.

Functions¶

identify_challenging_examples(task_description, input_prompt)¶

Generate challenging examples for the given task description.

Parameters:

  • task_description (str) – The task for which examples are to be generated.

  • input_prompt (str) – The prompt used to generate the challenging examples.

Returns:

List of challenging examples.

Return type:

list

annotate_challenging_examples(examples, input_prompt)¶

Annotate challenging examples with corresponding Python code generated from the LLM.

Parameters:

  • examples (list) – List of challenging examples.

  • input_prompt (str) – The prompt to generate code for the examples.

Returns:

List of dictionaries containing the example and its corresponding code.

Return type:

list of dict

annotate(annotations, initial_prompt)¶

Analyze and rank errors in the generated code for each example.

Parameters:

  • annotations (list) – List of dictionaries with challenging examples and their generated code.

  • initial_prompt (str) – The initial prompt used for code generation.

Returns:

List of examples with detailed error analysis and scores.

Return type:

list of dict

error_analysis_fun(input_prompt, annots)¶

Perform error analysis for the provided annotations and input prompt.

Parameters:

  • input_prompt (str) – The prompt used for generating code for challenging examples.

  • annots (list) – List of annotated examples with errors.

Returns:

Detailed analysis of errors.

Return type:

str

calibrate_generation_prompt(input_prompt, history, error_analysis, task_desc, meta_prompt)¶

Refine the input prompt based on task details, history, and error analysis.

Parameters:

  • input_prompt (str) – The prompt to be refined.

  • history (list) – A list of previously generated prompts.

  • error_analysis (list) – A list of identified errors.

  • task_desc (str) – A description of the task.

  • meta_prompt (str) – A base prompt used for code generation.

Returns:

Refined prompt.

Return type:

str

autoprompt(task_description, num_iter)¶

Iteratively refine the base meta prompt for the given task description.

Parameters:

  • task_description (str) – A description of the task to refine the prompt for.

  • num_iter (int) – The number of iterations to refine the prompt.

Returns:

The final refined prompt.

Return type:

str

Recommended Usage:

refined_prompt = autoprompt("Generate Python code for data cleaning tasks", 5)

Error Handling¶

The Autoprompt system includes robust error handling for:

  • Module installation errors during code execution.

  • Tool calling errors for generating incorrect responses.

  • Challenging example analysis failures.

Requirements¶

  • re

  • subprocess

  • traceback
